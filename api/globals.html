installMessageFilter

  • installMessageFilter(handler: IMessageHandler, filter: IMessageFilter): void
    • Defined in index.ts:206

    Install a message filter for a message handler.

    A message filter is invoked before the message handler processes a message. If the filter returns true from its filterMessage method, no other filters will be invoked, and the message will not be delivered.

    The most recently installed message filter is executed first.

    Parameters

    • handler: IMessageHandler

      The handler whose messages should be filtered.

    • filter: IMessageFilter

      The filter to install for the handler.

      Notes

      It is possible to install the same filter multiple times. If the filter should be unique, call removeMessageFilter first.

      See Also removeMessageFilter.

    Returns void

postMessage

  • postMessage(handler: IMessageHandler, msg: Message): void
    • Defined in index.ts:149

    Post a message to the message handler to process in the future.

    Parameters

    • handler: IMessageHandler

      The handler which should process the message.

    • msg: Message

      The message to post to the handler.

      Notes

      Unlike sendMessage, postMessage will schedule the deliver of the message for the next cycle of the event loop. The handler will have the opportunity to compress the message in order to optimize its handling of similar messages. The message will be sent through any installed message filters before being delivered to the handler.

      See Also sendMessage.

    Returns void

      The handler for which the filter is installed.

    • filter: IMessageFilter

      The filter to remove.

      Notes

      This will remove all occurrences of the filter. If the filter is not installed, this is a no-op.

    Send a message to the message handler to process immediately.

      The handler which should process the message.

    • msg: Message

      The message to send to the handler.

      Notes

      Unlike postMessage, sendMessage delivers the message to the handler immediately. The handler will not have the opportunity to compress the message, however the message will still be sent through any installed message filters.
